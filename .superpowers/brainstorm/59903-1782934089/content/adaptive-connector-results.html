Rendering Rules
1. Table

Use when output has arrays of objects: issues, PRs, tickets, users, logs, files.

2. Cards

Use when output has few rich records with descriptions, links, and metadata.

3. Summary Text

Use for narrative answers, single values, or tool-generated explanations.

4. JSON Inspector

Fallback for unknown shapes, with collapsible raw output so users can debug.

Error / zero result

Show query, connector, auth/search status, and next action. Do not silently say “0”.

A

Shape-Based Renderer

Recommended. Detect rows, cards, text, errors, and JSON from any connector output.

B

Connector-Specific Views

Polished for Jira/GitHub/etc., but every connector needs custom UI and maintenance.

C

LLM Markdown Summary

Flexible, but weak for accurate counts, tables, sorting, copy/export, and debugging.
