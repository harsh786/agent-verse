## Agent Architecture Patterns by Domain Type

### High-Volume Repetitive (GST, CA Firms, Invoicing, E-Commerce Catalog)
```
Celery Beat scheduled trigger (daily/monthly)
    │
Bulk processing: asyncio.gather (parallel execution for 50+ clients/items)
    │
Per-item: ERP/Portal MCP connector → Validate → Process → Verify
    │
HITL checkpoint for exceptions → Post results
```

### Multi-Step Approval-Gated (Legal, Banking, Construction)
```
User/system goal submitted
    │
Planner decomposes into multi-step plan with HITL nodes identified
    │
Execute steps 1..N → HITL gate (email/WhatsApp/Slack approval)
    │
Resume with approval context → Complete remaining steps
```

### Real-Time Monitoring + Event Response (DevOps, Cybersecurity, Telecom)
```
Celery Beat (every 30s–60min): Monitor → Detect anomaly
    │
Anomaly detected → Create incident goal → AgentGraph executes response
    │
Resolution verified → Post-mortem → Return to monitoring
```

### Research + Document Generation (Legal, CA Firms, Pharma, Construction)
```
User goal (e.g., "Prepare due diligence report")
    │
Parallel sub-agents: web search + document parse + database query
    │
Synthesize findings → Generate structured output
    │
HITL review → Deliver via email + store in knowledge base
```

## Quick Start

```bash
# Browse available domain configs
ls docs/domains/

# Pick a domain and read the use cases
cat docs/domains/27-accounting-ca-firm/use-cases.md

# Deploy with domain-specific config
uv run agentverse deploy \
  --manifest docs/domains/27-accounting-ca-firm/gst-filing-agent.yaml \
  --tenant your-tenant-id

# Submit a domain-specific goal
curl -X POST http://localhost:8000/api/goals \
  -H "X-API-Key: your-api-key" \
  -d '{"goal": "File GSTR-1 for all 50 clients for March 2025", \
       "agent_id": "ca-firm-gst-agent"}'
```
