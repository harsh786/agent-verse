# Manufacturing & Industry 4.0
### *Autonomous intelligence on the shop floor — from sensor data to boardroom decisions without human intermediaries*

---

## Executive Summary

India's manufacturing sector contributes ₹35 lakh crore to GDP and is racing toward Industry 4.0, yet 78% of plants still operate with reactive maintenance practices that cost the sector ₹4.2 lakh crore annually in unplanned downtime. Quality defects account for a further ₹1.8 lakh crore in waste and recalls. AgentVerse bridges the gap between operational technology (OT) and information technology (IT) — deploying agents that ingest sensor data, ERP records, and quality systems to deliver autonomous predictive maintenance, root-cause analysis, production optimisation, and regulatory compliance — turning every machine into a decision-making node in a self-optimising factory.

### UC-1: Predictive Maintenance Alerting

**The Problem:** Unplanned equipment downtime costs Indian manufacturers an average of ₹8,000–₹2.5 lakh per hour depending on industry. A mid-size automotive plant losing 4 hours/month to unexpected breakdowns wastes ₹12–₹40 crore annually. Traditional preventive maintenance based on fixed time intervals wastes 30% of maintenance spend on components that have remaining useful life, while still missing 45% of failures.

**AgentVerse Solution:** The agent continuously ingests vibration, temperature, pressure, and current sensor telemetry from IoT gateways, applies ML anomaly detection and remaining useful life (RUL) models, generates maintenance work orders in the CMMS before failure occurs, procures required spare parts automatically when inventory is low, and schedules maintenance during planned downtime windows — shifting the plant from reactive to predictive posture.

**Agent Workflow:**
1. IoT gateway streams sensor data (OPC-UA / MQTT) every 30 seconds via HTTP MCP connector.
2. Agent applies real-time rule-based anomaly checks: vibration > 12mm/s RMS, temperature > 85°C bearing, current draw 15% above baseline.
3. For subtle trend anomalies: code sandbox runs LSTM time-series model on 72-hour rolling window per asset.
4. Compute Remaining Useful Life (RUL) estimate: predict time-to-failure distribution for flagged assets.
5. If RUL < 14 days: generate predictive maintenance alert; classify fault mode (bearing wear, imbalance, misalignment, thermal stress).
6. Query CMMS (SAP PM / Maximo MCP connector): check if asset has scheduled maintenance within RUL window.
7. If no maintenance scheduled: create work order in CMMS with fault description, recommended corrective action, asset history summary.
8. Check spare parts inventory in ERP: if required spares stock < reorder point, auto-raise purchase requisition to preferred vendor.
9. Route work order to maintenance supervisor via Slack with priority (P1/P2/P3) and supporting sensor charts (chart image generated by code sandbox).
10. HITL gate: supervisor reviews P1 alerts (< 48 hours RUL); approves scheduling adjustment to accelerate maintenance.
11. Post-maintenance: agent validates that sensor readings return to normal range; closes work order loop.
12. Monthly: predictive maintenance KPI report — alerts raised, failures prevented, cost avoidance calculated, false-positive rate.

**Tools Used:** MQTT/OPC-UA IoT connector, Code execution sandbox (LSTM, RUL models, chart generation), CMMS API (SAP PM/Maximo), ERP MCP connector, Slack, HITL approval gate, PostgreSQL (asset history), Celery scheduler

**Revenue Model:** ₹8,00,000/month per 50-asset monitoring deployment; ₹25,00,000/month for full-plant (500+ assets); ROI-sharing pilot: 15% of documented cost avoidance

**ROI:** Unplanned downtime reduced 68%; maintenance cost reduced 23% (elimination of unnecessary time-based interventions); spare parts inventory carrying cost down 30%; payback in 4 months

**Target Customers:** Automotive OEMs, steel mills, cement plants, paper mills, chemical plants, food processing facilities

### UC-2: Quality Control Defect Root Cause Analysis

**The Problem:** A 1% defect rate costs a ₹500 crore/year manufacturer ₹5 crore in direct scrap and rework, plus ₹12–₹25 crore in warranty claims and recall risk. Quality engineers spend 60–70% of their time manually correlating defect data across machine parameters, raw material lots, operator shifts, and environmental conditions — a process that takes 3–7 days and still yields inconclusive results 35% of the time.

**AgentVerse Solution:** The agent automatically correlates defect occurrence data with production parameters from MES, raw material certificates from suppliers, machine sensor data, environmental readings, and operator shift records — running statistical correlation and causal inference algorithms to identify root causes in hours rather than days. It generates structured 8D/CAPA reports ready for customer submission.

**Agent Workflow:**
1. Defect detection trigger: QMS alert (failed inspection), customer complaint API, or automatic defect counter from vision system.
2. Agent collects defect data: part number, defect type, quantity, inspection date/time, machine ID, operator ID, shift.
3. Fetch production parameters from MES for the defect time window: machine speed, temperature, pressure, feed rate, cycle time.
4. Fetch raw material lot data from ERP: supplier, lot number, certificate of analysis (CoA) values — tensile strength, hardness, chemical composition.
5. Fetch environmental data: ambient temperature, humidity from building management system API during production window.
6. Retrieve operator qualification records and recent training log from HR system.
7. Code sandbox: run statistical correlation analysis (Pearson, Spearman) — identify parameters with correlation > 0.65 to defect occurrence.
8. Apply Bayesian causal inference to distinguish correlation from causation; generate probability ranking of root causes.
9. Cross-reference against historical defect database (pgvector similarity search): find similar past defect events and their resolved root causes.
10. Draft 8D problem-solving report: D1–D8 sections pre-filled with agent-gathered evidence; highlight most likely root cause (with confidence %).
11. HITL gate: quality engineer reviews AI-generated 8D; confirms or modifies root cause assessment; approves CAPA plan.
12. Send finalised 8D to customer (if external complaint) via email; update QMS with CAPA items and deadlines.

**Tools Used:** QMS API, MES API, ERP MCP connector, BMS API (environmental), HR system API, Code execution sandbox (stats, causal inference), PostgreSQL + pgvector (defect history), HITL approval gate, Email/SMTP, PDF generator

**Revenue Model:** ₹3,00,000/month quality intelligence module; ₹1,50,000 per root cause investigation engagement

**ROI:** Root cause identification time from 5 days to 6 hours; CAPA closure rate improved 58%; defect recurrence reduced 71%; warranty claim cost reduced ₹8–₹18 crore/year for automotive supplier

**Target Customers:** Auto-component manufacturers, electronics assemblers, pharmaceutical manufacturers, food processing companies, aerospace suppliers

### UC-3: Production Schedule Optimization

**The Problem:** Manual production scheduling in Indian SME/mid-market manufacturers achieves 58–65% schedule adherence due to ad-hoc changes, rush orders, resource conflicts, and poor constraint visibility. Each schedule change costs ₹25,000–₹2 lakh in setup, sequencing, and opportunity cost. Master Production Schedulers spend 4–6 hours daily on reactive replanning rather than strategic capacity planning.

**AgentVerse Solution:** The agent ingests the full demand picture (confirmed orders, forecasts, seasonal commitments), resource constraints (machine capacity, tooling availability, operator skills, raw material stock), and business rules (priority customers, minimum batch sizes, changeover sequences), and generates an optimised production schedule — automatically replanning when disruptions occur (machine breakdown, rush order, material shortage) in under 10 minutes.

**Agent Workflow:**
1. Daily at 18:00, agent fetches current order book from ERP: confirmed orders, quantities, delivery dates, priority flags.
2. Fetch open purchase orders and expected material arrival dates (ERP) — identify material-constrained slots.
3. Pull machine availability calendar from CMMS: scheduled maintenance windows, machine capacity per shift.
4. Fetch operator roster and skill matrix from HR/MES system: who is available, which machines they are qualified for.
5. Code sandbox: formulate as Mixed Integer Programming problem (OR-Tools solver); minimise makespan + setup time + late deliveries, subject to all constraints.
6. Generate optimised schedule: machine-wise Gantt chart (hour-level precision) for 14-day rolling horizon.
7. Identify scheduling conflicts: material arrival after required start date, capacity bottleneck, operator gap.
8. For each conflict: generate alternative resolution scenarios (expedite material / outsource / push delivery date / overtime).
9. HITL gate: production manager reviews schedule via Slack-embedded Gantt chart summary; approves or selects conflict resolutions.
10. Post-approval: push schedule to MES via API; trigger work order release for next 48-hour execution window.
11. Real-time disruption monitoring (Celery): if breakdown alert or rush order received during execution, auto-trigger replanning for remaining horizon.
12. Weekly adherence report: planned vs. actual schedule, disruption frequency, on-time delivery performance, machine utilisation.

**Tools Used:** ERP MCP connector, CMMS API, HR/MES API, Code execution sandbox (OR-Tools MIP solver, Gantt chart generation), Slack (HITL Gantt review), PostgreSQL (schedule history), Celery (disruption monitoring), MES API

**Revenue Model:** ₹2,50,000/month scheduling optimisation module; ₹8,00,000/month full plant suite

**ROI:** Schedule adherence from 62% to 87%; on-time delivery to customer improved 28%; machine utilisation up 14%; planner productivity 4× (reactive time reduced from 5 to 1.2 hours/day)

**Target Customers:** Discrete manufacturers (auto, electronics, industrial equipment), process manufacturers (pharma, food), garment manufacturers

### UC-4: Supplier Quality Monitoring & Scorecard

**The Problem:** A single bad batch of raw material can cause 10,000+ defective parts in an automotive or electronics assembly line, triggering customer line-stoppages and warranty costs of ₹2–₹20 crore per incident. Yet 67% of Indian manufacturers lack systematic supplier quality data aggregation — decisions are based on anecdote rather than evidence. IATF 16949 and ISO 9001 require documented supplier performance records that most manufacturers maintain only on spreadsheets.

**AgentVerse Solution:** The agent aggregates incoming quality inspection data, delivery performance metrics, CoA deviations, and customer complaint data attributable to each supplier, computes a multi-dimensional quality scorecard, triggers corrective action requests for underperforming suppliers, and publishes the scorecard to a supplier portal — creating an objective, audit-ready supplier quality management system.

**Agent Workflow:**
1. Daily: agent fetches incoming inspection results from QMS for all received materials in past 24 hours.
2. Parse CoA (PDF) from each delivery: extract actual values for specified parameters; compare against engineering specification limits.
3. Calculate incoming quality metrics per supplier: lot acceptance rate, parts-per-million (PPM) defect rate, major vs. minor defects.
4. Fetch delivery performance from ERP: planned vs. actual delivery date per PO; compute on-time delivery %.
5. Attribute customer complaints (from QMS) to root-cause supplier when traceable via lot/batch number.
6. Compute weighted supplier quality score: incoming quality 40%, delivery 30%, complaint attributions 20%, CoA accuracy 10%.
7. Classify supplier into tier: Approved (score > 85), Conditional (65–85), Probationary (50–65), Disqualified (< 50).
8. For Conditional/Probationary suppliers: auto-generate Supplier Corrective Action Request (SCAR) document with specific evidence.
9. Send SCAR via email (SMTP) to supplier quality contact; set 30-day response deadline; log in SCAR tracker (PostgreSQL).
10. HITL gate: Supplier Quality Engineer reviews Disqualified supplier cases; approves disqualification or escalated action.
11. Monthly: publish scorecard to supplier portal (PDF report per supplier via email); update approved supplier list in ERP.
12. Quarterly: aggregate trends report for management — supplier quality cost of poor quality (COPQ), top 5 problem suppliers, improvement actions status.

**Tools Used:** QMS API, ERP MCP connector, PDF parser (CoA extraction), Code execution sandbox (scoring, statistics), Email/SMTP, PostgreSQL (SCAR tracker, scorecard DB), HITL approval gate, PDF generator (scorecard, SCAR), Slack, Celery scheduler

**Revenue Model:** ₹1,80,000/month supplier quality module; ₹3,00,000/month with supplier portal access for up to 100 suppliers

**ROI:** Incoming PPM defects reduced 52% (within 12 months of SCAR programme); customer line-stoppage incidents reduced 73%; IATF/ISO audit finding on supplier records eliminated; ₹6 crore warranty cost reduction

**Target Customers:** IATF 16949-certified auto-component companies, ISO 13485 medical device manufacturers, aerospace Tier-1 and Tier-2 suppliers

### UC-5: Safety Incident Reporting & Compliance

**The Problem:** India reports 48,000+ occupational fatalities annually (MoLE data); non-fatal injuries in manufacturing are under-reported by 60–70%. Companies with poor EHS compliance face Factory Act penalties (₹1–₹5 lakh per violation), OSHA-equivalent state-level penalties, and — critically — criminal liability under Factories Act Section 92. ISO 45001 certification maintenance requires meticulous incident investigation and CAPA records that most manufacturers maintain inconsistently.

**AgentVerse Solution:** The agent digitises the full safety incident lifecycle — from first report through root cause investigation, CAPA implementation, and statutory reporting to the Factory Inspector — ensuring zero reporting gaps, consistent investigation quality, and audit-ready documentation. It also monitors near-miss trends to prevent incidents before they occur.

**Agent Workflow:**
1. Safety incident reported: mobile app, QR-code scan form, or email notification triggers agent.
2. Immediate triage: classify severity — Fatality / Lost-Time Injury (LTI) / Recordable / Near Miss / Unsafe Condition.
3. For LTI and above: immediately notify Plant Head, EHS Manager, HR Head, and Legal Head via Slack + email (within 5 minutes of report).
4. HITL gate: EHS Manager confirms notification receipt; assigns investigation team from approved investigator roster.
5. Agent creates investigation task checklist: witness interviews required, CCTV footage preservation, equipment lockout/tagout verification, medical report collection.
6. Compile preliminary incident report (PDF) for Factory Inspector filing: required within 24 hours under Factories Act for LTI+.
7. RPA: log incident on DGFASLI portal / State Factory Inspectorate online portal within statutory deadline.
8. As investigation progresses: agent ingests uploaded evidence (photos, CCTV timestamps, witness statements) via document portal.
9. Code sandbox: run 5-Why analysis and Fishbone template auto-population based on evidence collected.
10. Draft CAPA action plan: for each root cause, agent suggests corrective action, assigns responsible person, sets deadline.
11. Celery scheduler monitors CAPA completion: sends reminders to responsible persons; escalates overdue CAPAs to Plant Head.
12. Monthly EHS KPI dashboard: LTIFR, TRIFR, near-miss trend, CAPA completion rate, risk heat map by department.

**Tools Used:** Mobile form API, Email/SMTP, Slack, HITL approval gate, PDF generator, Playwright RPA (DGFASLI/State portal), Code execution sandbox (5-Why, Fishbone templates), Document storage, PostgreSQL (incident DB, CAPA tracker), Celery scheduler

**Revenue Model:** ₹1,50,000/month safety management module; ₹80,000 per ISO 45001 audit prep engagement

**ROI:** Factory Act penalties avoided (₹50 lakh–₹2 crore/year risk); LTIFR improved 45% within 12 months; incident investigation cycle from 14 days to 5 days; ISO 45001 audit finding rate reduced 82%

**Target Customers:** Large manufacturing plants, chemical complexes, construction companies, mining operations, food processing facilities

### UC-6: BOM Management & Costing

**The Problem:** Bill of Materials (BOM) management is the bane of complex product manufacturers: a single product family may have 500–2,000 BOM variants; engineering change orders (ECOs) must cascade across all variants; and cost impact of each ECO must be calculated before release. Manual ECO processing takes 5–15 days; costing errors on BOMs cost manufacturers 3–8% of product margin through undetected over-costing or under-pricing.

**AgentVerse Solution:** The agent automates ECO impact propagation across the BOM tree, re-calculates standard costs using the latest vendor pricing and labour rates, identifies costing anomalies, and publishes updated BOMs to the ERP with a full change audit trail — compressing ECO cycle time from 12 days to under 6 hours for standard changes.

**Agent Workflow:**
1. ECO raised in PLM/PDM system triggers agent via webhook.
2. Agent fetches affected part record from ERP BOM structure: identify all parent assemblies where the changed part is used (where-used report).
3. Validate change: check if new part specification meets engineering drawing requirements; flag mismatches.
4. For each affected parent BOM: fetch current BOM structure (multi-level explosion from ERP BOM API).
5. Apply ECO change: substitute old part with new part at all applicable BOM levels; update quantities/operations if changed.
6. Fetch latest purchase price for new/modified components from vendor price list (ERP procurement module) or via vendor portal RPA.
7. Code sandbox: recalculate standard cost for each affected finished good BOM — direct material + direct labour (using routings) + overhead allocation.
8. Generate cost impact analysis: old cost vs. new cost per product; identify products where margin drops below threshold.
9. For margin-impact > 2%: flag for pricing review — generate report for Finance and Commercial teams.
10. HITL gate: Engineering Head and Finance Controller approve ECO for ERP release.
11. Post-approval: update ERP BOMs via API; set effective date; create supersession record for old part.
12. Notify manufacturing, purchasing, and planning teams via Slack of BOM effective date and key changes.

**Tools Used:** PLM/PDM API (PTC Windchill/SOLIDWORKS PDM webhook), ERP MCP connector (BOM API), Code execution sandbox (BOM costing calculations), Playwright RPA (vendor portals), HITL approval gate, Slack, Email/SMTP, PostgreSQL (change audit trail), PDF generator

**Revenue Model:** ₹2,00,000/month BOM management module; ₹5,00,000/month PLM + ERP integration suite

**ROI:** ECO cycle from 12 days to 6 hours; costing errors reduced 88%; engineering change backlog cleared 70% faster; margin leakage from under-costed products recovered ₹3–₹12 crore/year

**Target Customers:** Automotive Tier-1 suppliers, industrial equipment OEMs, electronics manufacturers, aerospace component makers

### UC-7: Energy Consumption Optimization

India's industrial electricity rates have risen 42% over 5 years. Most plants have no system to correlate production output with energy consumption, making the ₹4–₹40 crore annual energy bill essentially unmanaged. Bureau of Energy Efficiency (BEE) mandates energy audits and Perform-Achieve-Trade (PAT) compliance for designated consumers.

**AgentVerse Solution:** The agent continuously monitors energy consumption from smart meters and sub-meters across all production areas and utilities, correlates energy vs. output using normalised energy intensity metrics, identifies waste patterns (idle equipment drawing power, power factor penalties, off-peak vs. on-peak consumption mix), and automatically acts on opportunities — shutting down equipment during breaks, shifting loads to off-peak tariff windows, and generating BEE-compliant energy audit reports.

**Agent Workflow:**
1. Smart meter data streams every 15 minutes via energy management system API into AgentVerse.
2. Agent maintains real-time energy intensity metrics: kWh per unit produced per production line.
3. Detect anomalies: equipment consuming > 20% above baseline energy for production level (code sandbox statistical model).
4. Identify non-production consumption: energy draw during planned stoppages (break time, shift change) — flag as avoidable waste.
5. Monitor power factor (PF) by feeder: if PF drops < 0.90, generate alert to electrical team (poor PF attracts penalty from DISCOM).
6. Analyse tariff optimisation: current on-peak vs. off-peak consumption ratio; model shift of shiftable loads (curing ovens, compressors) to off-peak.
7. For actionable savings: generate work order (CMMS) for electrical team — capacitor bank adjustment, compressed air pressure reduction, HVAC scheduling.
8. Celery automation: at shift break signal (from MES), auto-command non-critical equipment to standby mode via BMS API.
9. Monthly: generate SEC (Specific Energy Consumption) trend report vs. target; PAT compliance status.
10. HITL gate: energy manager reviews significant automation actions (HVAC changes affecting production areas) before execution.
11. Quarterly: auto-generate BEE energy audit report (PDF) — consumption by end-use, savings identified, investment ROI table.
12. Annual: compile PAT compliance data for submission to BEE portal (RPA on BEE Perform Achieve Trade portal).

**Tools Used:** Energy management system API, Smart meter MCP connector, BMS API, MES API, Code execution sandbox (energy models, regression), CMMS API, Playwright RPA (BEE portal), HITL approval gate, Slack, Email/SMTP, PDF generator, Celery scheduler

**Revenue Model:** ₹2,50,000/month energy optimisation module; success-fee option: 15% of documented energy savings

### UC-8: OEE Reporting & Analysis

**The Problem:** Overall Equipment Effectiveness (OEE) is the gold standard KPI for manufacturing productivity, yet only 12% of Indian manufacturers calculate OEE in real time. Manual OEE calculation from paper-based production logs takes 2–3 days per week and is 15–25% inaccurate due to data entry errors and inconsistent loss categorisation. World-class OEE is 85%; the average Indian SME manufacturer achieves 48–58%.

**AgentVerse Solution:** The agent collects real-time production counts and downtime data from MES/PLC systems, automatically categorises every stoppage event against the industry-standard loss tree (planned maintenance, unplanned breakdown, changeover, speed loss, quality rejection), calculates Availability × Performance × Quality in real time, and publishes live OEE dashboards and structured analysis reports — enabling daily management meetings to be data-driven rather than anecdotal.

**Agent Workflow:**
1. MES / PLC signals stream production counter, downtime events, and speed data every 60 seconds.
2. Agent accumulates data per shift: total production time, planned downtime, unplanned stoppages with duration and reason code.
3. For stoppages without reason code (operator forgot to enter): query MES history + maintenance system; infer probable cause category.
4. Calculate Availability: (Available Time − Downtime) / Available Time per machine.
5. Calculate Performance: (Actual Output × Ideal Cycle Time) / Available Time.
6. Calculate Quality: (Good Parts Produced) / (Total Parts Produced) — link to QMS rejection data.
7. OEE = Availability × Performance × Quality; compute per machine, per line, per shift, per plant.
8. Waterfall loss analysis: quantify each loss category in units, time, and revenue equivalent.
9. Identify top-3 OEE improvement opportunities by loss value; generate focused improvement recommendation (action, expected gain, responsible team).
10. Publish real-time OEE to plant TV dashboard (API push to digital signage system) and Slack `#production-floor`.
11. End-of-shift: generate shift OEE report PDF; email to shift manager and production superintendent.
12. Monthly: trend analysis — OEE trend vs. target, loss category trends, line-level comparison, top improvement project progress.

**Tools Used:** MES/PLC API, QMS API, Code execution sandbox (OEE calculations, waterfall chart generation), Digital signage API, Slack, Email/SMTP, PDF generator, PostgreSQL (historical OEE database), Celery scheduler

**Revenue Model:** ₹1,50,000/month OEE reporting module; ₹3,50,000/month with loss elimination consulting recommendations

**ROI:** OEE improvement from 56% to 71% in 6 months (₹8–₹40 crore revenue uplift per shift from freed capacity); management meeting quality improved; reactive maintenance reduced 35% from early signal detection

**Target Customers:** Any discrete or process manufacturer; automotive OEMs; FMCG plants; pharmaceutical manufacturers; electronics assembly

### UC-10: Supply Chain Disruption Response

**The Problem:** The COVID-19 pandemic exposed Indian manufacturers' fragility to supply chain shocks, with 78% experiencing production stoppages due to single-source dependencies. Geopolitical disruptions (chip shortage, port congestion) continue to cause ₹1.2 lakh crore in annual production losses for Indian industry. Manual disruption response takes 48–72 hours to assess impact and identify alternatives; by then, production lines are already stopped.

**AgentVerse Solution:** The agent continuously monitors supplier health signals (financial distress news, port congestion data, weather events, geopolitical alerts) and responds autonomously to confirmed disruptions — within 2 hours of signal: assessing inventory coverage, identifying alternative sources, computing make-vs.-buy options, and presenting a ranked action plan to supply chain leadership for rapid decision and execution.

**Agent Workflow:**
1. Celery monitors SearXNG news feeds every 2 hours: keywords including key supplier names, raw material categories, port names, geopolitical risk zones.
2. Detect disruption signals: supplier financial distress article, port congestion advisory, force majeure declaration, factory fire/flood news.
3. Cross-reference affected supplier/raw material against supplier master and active open POs in ERP.
4. Compute inventory coverage: current stock + in-transit / average daily consumption per material = days of supply.
5. Identify vulnerable items: days of supply < 21 days from affected sources.
6. Query alternative supplier database (approved supplier list in ERP): identify alternate qualified sources per vulnerable material.
7. Check alternate supplier capacity and lead time via email inquiry (SMTP) or supplier portal (RPA) — automated request for availability confirmation.
8. Model substitute materials: check if engineering-approved substitutes exist in material master; confirm with engineering via Slack question.
9. Compute options matrix: continue with current supplier (risk), accelerate remaining deliveries, switch to alternate, expedite air freight, substitute material.
10. HITL gate: CPO/VP Supply Chain reviews options matrix (cost impact, risk reduction, lead time) on Slack approval workflow.
11. Execute approved option: raise emergency PO to alternate supplier, revise production schedule in MES, communicate to operations and finance.
12. Monitor recovery: daily status update from disrupted supplier via automated email check; close disruption record when supply normalised.

**Tools Used:** SearXNG (news monitoring), ERP MCP connector, PostgreSQL (inventory, alternate supplier DB), Email/SMTP (supplier inquiry), Playwright RPA (supplier portals), Code execution sandbox (scenario modelling), HITL approval gate, Slack, MES API, Celery scheduler
